# 📖 Staff Manual PWA System Specification

This document provides a comprehensive blueprint for the "Store Manual PWA" system. It is designed to allow another AI or developer to fully recreate the application with the same architecture and functionality.

---

## 🏗️ 1. System Architecture
- **Type**: Single Page Application (SPA) with Progressive Web App (PWA) capabilities.
- **Frontend**: Vanilla HTML5, Vanilla CSS3 (modern aesthetics), and Vanilla JavaScript (ES6+). No heavy frameworks (React/Vue) were used to ensure maximum speed and zero build-step overhead.
- **Backend**: Vercel Serverless Functions (Node.js) for API endpoints.
- **Database**: Vercel KV (Redis) for dynamic data (checklists/tasks).
- **AI**: Google Gemini 2.5 Flash API for the manual-aware chat assistant.
- **Deployment**: Vercel (Automatic deployment via GitHub).

## 🚀 4. Key Functional Logic

### 4.1 Content Rendering Engine
- Uses **Template Literals** in JS to inject `MANUAL_DATA` into the DOM.
- **Navigation**: [navigateTo(catId, sectionId)](file:///Users/takaragiryusei/Documents/yemolthouse/store-manual/store-manual/js/app_en.js#83-123) updates the URL-like state and swaps the innerHTML of the `#main-content` div.
- **Search**: Scans `MANUAL_DATA` using `query.includes()` and highlights results in a dropdown.

### 4.2 AI Chat Assistant (The "Smart" Manual)
- **Prompt Logic**: The `systemPrompt` includes the *entire* manual content as context.
- **Rules**: Strictly answers only from the manual; otherwise, refers to the manager.
- **API**: POST request to Vercel Serverless Function -> Gemini API.

### 4.3 Task Board & Persistence
- **Daily Reset**: Key is generated as `prefix:YYYY-MM-DD` (JST).
- **Storage**: Uses Vercel KV (Redis) with a 7-day TTL for auto-cleanup.
- **Isolation**: Each store uses a unique prefix (e.g., `fc_uber_tasks:`) to share a single DB.
- *Note: In the current version, this UI is commented out to allow deployment without a credit card (Vercel KV requirement).*

### 4.4 PWA & Offline Support
- **Dynamic Caching**: [sw.js](file:///Users/takaragiryusei/Documents/yemolthouse/store-manual/store-manual/sw.js) caches assets on first load.
- **Offline Mode**: Manual remains readable even without internet once visited.

## 🎨 5. UI/UX Design Principles
- **Aesthetic**: Premium "Glassmorphism" look.
- **Colors**: Uses CSS Variables (`--accent-primary`, `--bg-main`, etc.) for easy theme swaps.
- **Responsiveness**: Mobile-first design. Sidebar slides in/out on mobile.
- **Feedback**: Skeleton loaders and subtle micro-animations for interactions.

---

## ⚙️ 6. Setup & Environment Variables

To replicate this system, the following environment variables are required on Vercel:
1. `GEMINI_API_KEY`: API Key from Google AI Studio.
2. `KV_REST_API_URL`: Vercel KV endpoint.
3. `KV_REST_API_TOKEN`: Vercel KV secret token.

---

## 📝 7. Integration Guidelines for another AI
1. **Initialize**: Tell the AI to build a PWA using Vanilla JS and Vercel.
2. **Data-Driven**: Provide the `MANUAL_DATA` structure and ask it to build the rendering logic first.
3. **Prompt Engineering**: The AI Assistant and the user-facing AI Chat should share the manual content consistently.
4. **Offline First**: Ensure the Service Worker correctly handles asset caching.
